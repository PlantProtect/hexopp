2.3 SpringBoot 项目启动流程

  • LAPTOP-NMUAI4F7:主机号
  • PID:进程号

  • profiles:配置文件

  • 没有活跃的应用配置程序配置文件被设置。系统调用缺省default的配置文件来完成用户

  • web应用服务器（tomcat）启动，端口号8080
  • 当前web服务器通过8080端口向用户提供http服务
  • 表示tomcat启动成功
  • 无法找到Thymeleaf模板文件
  • Thymeleaf为用户提供模板页面呈现
  • 表明启动通过了

3.应用程序的配置文件 applacation.yaml

3.1 在resource目录下，创建applaciton.yaml

  • 默认端口号改为8081

  • 应用名设为study

  • 启动应用程序，控制台输出时：

  • 在浏览器输入

    • study 代表服务器上的study项目
    • application.yaml (实验二)

  • server

  • spring:

    • database //定义JDBC连接MYSQL数据库

      • driver-class-name：JDBC的驱动类

      • url：连接字符串

        • jdbc:mysql:协议名
        • localhost：MySQl数据库服务器的IP地址
        • 3306：安装数据库时的设置的端口号
        • test：目标数据库（test数据库）
        • serverTimezone：数据库的时区信息（GMT%2B8<== GMT 8,东8区)
        • characterEncoding：编码集UTF8,unicode编码
        • useSSL：是否使用SSL加密协议

      • username：用户名

      • root：密码

    • thymeleaf: //定义模板页面文字

      • cache：是否启动页面缓存（缓存，只能判断存在还是不存在，无法判断页面是否修改，所以在开发时一般关闭）F5是刷新，ctrl+F5是强制刷新，系统将服务器端所有资源重新下载。
      • encoding：thymeleaf页面的编码方式
      • mode：HTML5标准
      • suffix、prefix：后缀和前缀 ==> 对哪些文件启动Thymeleaf模板引擎
        • 对classpath:/templates/目录下，以.html结尾的文件执行thymeleaf模板
        • classes目录是编译后产生的文件，在target文件夹下（target是编译后的文件夹）
